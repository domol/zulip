# SafeChat Web Server ‚Äî Claude Code Guide

This is a white-labeled fork of [zulip/zulip](https://github.com/zulip/zulip),
the Zulip Django/Python web server + TypeScript frontend.

> **Note:** `.claude/CLAUDE.md` in this repo is Zulip's own contributor guide ‚Äî
> read it when making functional changes that may be submitted upstream.
> This file covers SafeChat-specific workflow only.

---

## Branch structure

```
upstream/main  ‚Üí  sync  ‚Üí  main (branded)
```

| Branch | Purpose |
|---|---|
| `upstream/main` | Mirror of zulip/zulip. Never commit here directly. |
| `sync` | All functional changes. **Zero branding changes.** |
| `main` | Production-ready, fully branded. Generated by `rebrand.sh` after merging sync. |

Remote `upstream` points to `git@github.com:zulip/zulip.git`.
Add your own fork as `origin` when ready:
```bash
git remote add origin git@github.com:YOUR-ORG/safechat.git
```

---

## Upstream merge workflow

```bash
git fetch upstream
git checkout sync
git merge upstream/main          # resolve conflicts here, no branding strings involved
git checkout main
git merge sync
./rebrand.sh
git add -A
git commit -m "rebrand: apply after upstream merge $(git describe upstream/main --tags)"
```

## Branding system

Brand values come from **`../branding/config.sh`** (shared root directory):
- `BRAND_NAME`, `BRAND_DOMAIN`, `BRAND_HELP_URL`, `BRAND_STATUS_URL`
- `BRAND_SUPPORT_EMAIL`, `BRAND_SALES_EMAIL`, `BRAND_ADMIN_NAME`

**`rebrand.sh`** applies them to:

| File | What changes |
|---|---|
| `templates/zerver/base.html` | Page title suffix `- Zulip` |
| `templates/zerver/portico-header.html` | Logo alt/aria-label |
| `zerver/lib/send_email.py` | Email `from_name` default |
| `zerver/lib/email_notifications.py` | `reply_to_name`, why-link |
| `zproject/default_settings.py` | `ADMINS` display name |
| `zerver/models/realms.py` | Default discussion channel name |
| `templates/500.html` | Status page URL |
| `web/src/tippyjs.ts` | Sales contact email |
| `web/src/navbar_alerts.ts` | Help centre link |
| `templates/zerver/emails/*.html|txt` | "Zulip organization/Cloud" phrases |
| `static/images/logo/`, `favicon.*`, `email_logo.png` | Brand assets |

### What rebrand.sh intentionally does NOT change

| Item | Reason |
|---|---|
| Django app names (`zerver`, `zilencer`, `analytics`) | Internal, all imports reference these |
| System user `zulip` in `puppet/` | OS-level, changing breaks deployment |
| Database migration files | Immutable history |
| `.claude/CLAUDE.md` | Zulip's contributor guide ‚Äî leave untouched |
| Client name detection (`if client_name.startswith("Zulip")`) | Identifies Zulip mobile clients |
| `corporate/` templates | Zulip.com marketing ‚Äî disable/delete separately |

## Deployment pipeline

Production runs on a standard Ubuntu Zulip installation (not Docker).

Push to `main` ‚Üí GitHub Actions (`.github/workflows/deploy.yml`):
1. `tools/provision --build-release-tarball-only` ‚Äî installs build deps (~15 min cold, ~3 min cached)
2. `tools/build-release-tarball safechat` ‚Äî compiles assets, creates tarball at `$OUTPUT_DIR/zulip-server-safechat.tar.gz`
3. SCP tarball to `/tmp/` on server
4. SSH ‚Üí `sudo upgrade-zulip /tmp/zulip-server-safechat.tar.gz`

Zulip's `upgrade-zulip` script handles everything: unpacks to `/home/zulip/deployments/<timestamp>/`, runs migrations, restarts supervisor workers and nginx, updates the `current` symlink.

**Secrets required in the GitHub repo:**
- `DEPLOY_HOST` ‚Äî server hostname or IP
- `DEPLOY_USER` ‚Äî SSH user with passwordless sudo for upgrade-zulip
- `DEPLOY_SSH_KEY` ‚Äî ed25519 private key (public key in server's authorized_keys)

**Rollback:** previous deployments stay at `/home/zulip/deployments/`; run `upgrade-zulip` with the old directory path.

## School-specific configuration

Zulip has extensive per-realm configuration. For SafeChat deployments:

- Set `REALM_CREATION_LINK_VALIDITY_DAYS` appropriately for school onboarding
- Consider disabling `allow_logo_upload` in default settings if you want consistent branding
- The `corporate/` app (Zulip.com billing/marketing) should be disabled or removed
- Review `zerver/models/realms.py` `PlanType` choices ‚Äî SafeChat likely wants a fixed plan

## Development notes

- `./tools/run-dev` starts the dev server
- `./tools/test-backend` runs Python tests
- `./tools/test-js-with-node` runs JS tests
- `./tools/lint` runs all linters including mypy
- User-facing strings in Python must use `_()` for translation
- Locale `.po` files in `locale/` are generated from source ‚Äî do not edit directly
